# 语音引导与配置面板一致性改造计划（2026-03-14）

## 背景

当前首次引导页（ASR/TTS 步骤）与配置面板（VoiceSettingsPanel）共用同一套底层配置存储（`desktopBridge.settings.get/save` + 主进程 `settingsStore`），但表单实现分叉，导致体验与维护不一致：

- 引导页云端模型字段是自由输入。
- 配置面板云端模型字段是下拉清单（并兼容历史自定义值）。
- 引导页存在状态显示错配：`skip` 状态被显示为 `cloud`，但配置区仍按 `skip` 渲染，造成“显示云端却不展示云端配置”的错觉。

## 目标

- 统一首次引导与配置面板的语音配置交互语义与模型来源。
- 将 ASR/TTS 都改为三段式流程：`选择来源与模型` -> `填写配置` -> `测试`。
- 保证引导页与配置面板修改后行为一致，避免“改一处漏一处”。

## 范围

- 首次引导：`front_end/src/components/onboarding/FirstRunOnboardingDialog.jsx`
- 配置面板：`front_end/src/components/config/VoiceSettingsPanel.jsx`
- 前端设置桥接：`front_end/src/services/desktopBridge.js`
- 主进程设置存储：`desktop/electron/services/settingsStore.js`
- 国际化文案：`front_end/src/i18n/I18nContext.jsx`

## 现状问题与根因

1. 引导页来源显示与状态值不一致  
   - 问题：ASR 来源下拉将 `skip` 显示成 `cloud`，但云端配置区仅在 `asrSource === 'cloud'` 时渲染。  
   - 根因：`value={asrSource === 'skip' ? 'cloud' : asrSource}` 的显示映射与真实状态不一致。

2. 云端模型输入策略不一致  
   - 引导页：ASR/TTS 云端模型自由输入。  
   - 配置面板：ASR/TTS 云端模型下拉清单（可带历史自定义项）。

3. 可维护性风险  
   - 同一配置字段有两套 UI 规则，后续新增模型、改字段约束、改默认值时容易漏改。

## 统一方案

1. ASR/TTS 引导步骤改为三段式
   - 步骤 A：来源选择 + 具体模型选择（两级下拉，必须显式选定）。
   - 步骤 B：配置填写（仅展示当前来源/模型需要的字段）。
   - 步骤 C：测试并反馈（延迟、识别文本、首包时间、错误原因）。

2. 来源与模型选择规则
   - ASR：
     - 来源：云端（DashScope）/ 本地。
     - 具体模型：来源为云端时展示 DashScope ASR 模型清单；来源为本地时展示本地内置模型清单。
   - TTS：
     - 来源：云端（DashScope）/ 云端（免 Key）/ 本地。
     - 具体模型：根据来源展示对应模型清单。

3. 模型清单单一来源
   - 将 DashScope ASR/TTS 模型选项收敛为共享常量（或共享模块），引导页与配置面板共同引用，禁止两套硬编码列表漂移。
   - 历史自定义值策略：
     - 默认允许“只读可见 + 明确标记为自定义”，并引导用户切换到正式清单项。
     - 未来可评估是否完全禁止新写入自定义模型。

4. 状态一致性修复
   - 移除 `skip -> cloud` 的显示映射。
   - 下拉值、条件渲染、保存行为必须全部使用同一个状态值。

5. 字段展示最小化
   - 仅当来源和模型需要时展示字段：
     - DashScope：API Key、Base URL、Workspace、语言/音色/采样率等。
     - 本地模型：下载/重下/应用操作与下载状态卡片。
   - API Key 统一使用密码掩码显示策略（与配置面板保持一致）。

## 配置一致性原则

- 单一真值源：`settingsStore.voice` 为唯一配置真值。
- 单一写入口：前端统一通过 `desktopBridge.settings.save` 写入。
- 单一读入口：前端统一通过 `desktopBridge.settings.get` 回填。
- 引导页与配置面板只允许“展示层差异”，不允许“字段语义差异”。

## 验收标准

- 引导页 ASR/TTS 不再出现“显示云端但不显示配置区”的状态错配。
- 引导页与配置面板在同一字段上展示一致（来源、模型、API Key、语言/音色等）。
- 云端模型选择在两处都基于同一清单；新增或下线模型时只需改一处定义。
- 用户在任一入口保存后，另一个入口打开时字段值与生效行为一致。

## 回归测试清单

- 引导页 ASR：来源切换（云端/本地）-> 模型切换 -> 应用 -> 延迟测试。
- 引导页 TTS：来源切换（云端/免 Key/本地）-> 模型切换 -> 应用 -> 延迟测试。
- 配置面板 ASR/TTS：同样流程回归，验证与引导页一致。
- 跨入口一致性：引导页保存后进配置面板核对；配置面板保存后回到引导页核对。
- 历史配置兼容：已有自定义模型值时，UI 有明确提示且不崩溃。

## 风险与控制

- 风险：模型清单维护频率高，易过期。  
  - 控制：统一清单模块 + 发布前 checklist 校验。
- 风险：历史自由输入值被严格下拉限制后不可回显。  
  - 控制：保留“自定义项回显”兼容策略，并提供迁移提示。
- 风险：三段式流程增加点击次数。  
  - 控制：仅在首次引导使用三段式；配置面板保持单页但分区清晰。
