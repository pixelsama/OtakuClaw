# Free-Agent-Vtuber-openclaw 项目改进与功能规划（2026-03-02）

## 背景
当前项目主链路已经切换为 Electron + OpenClaw 文本流，核心能力可用。
本文件聚焦两部分：
1. 主要改进点（稳定性、安全性、工程质量）
2. 可增加功能（对标 Open-LLM-VTuber-Web 并结合当前架构可落地）

---

## 一、主要改进点（按优先级）

### P0
1. 修复 output-handler 的超时机制失效风险
- 问题：当前超时判断依赖 `pubsub.listen()` 循环进入，若无消息可能长时间阻塞，导致连接挂起。
- 影响：前端等待无反馈，任务长时间占用连接。
- 建议：改为可轮询超时模型（如 `get_message(timeout=...)`）或增加独立 watchdog 任务。

2. 改善敏感信息存储方式（OpenClaw Token）
- 问题：Token 当前会以明文落盘（Electron 设置文件）或存入浏览器 localStorage（Web fallback）。
- 影响：本地泄漏风险较高。
- 建议：
  - Electron 侧：使用系统密钥链（如 keytar）存储 token；
  - 设置文件只保留非敏感字段（baseUrl、agentId）；
  - Web fallback 仅用于开发，并增加明显风险提示。

### P1
3. Electron 安全硬化
- 问题：BrowserWindow 当前 `sandbox: false`。
- 影响：渲染进程被攻击时的隔离能力不足。
- 建议：开启沙箱、最小化 preload 暴露 API、持续保持 `contextIsolation: true` 且避免扩展桥接权限。

4. Web fallback 的 SSE 兼容性改进
- 问题：前端 SSE 分块解析主要按 `\n\n` 分隔，遇到 `\r\n` 风格流可能出现解析异常。
- 影响：跨网关/跨平台时流式内容可能中断或卡住。
- 建议：先统一换行再解析，并补充对应单元测试。

5. 文档与项目现状统一
- 问题：仓库内仍有旧技术栈/旧运行方式描述（如 Vue、全栈 compose），与当前 Electron 主链路不一致。
- 影响：新成员接入成本高，操作易出错。
- 建议：
  - 统一 README、AGENTS、docs；
  - 明确“当前推荐启动方式”和“历史服务状态”；
  - 把历史方案集中放入 archive 并加说明。

6. 测试覆盖率补齐
- 问题：前端测试目前仅 smoke test，关键链路缺少回归保护。
- 建议优先补测：
  - IPC 流式事件映射（`text-delta/done/error`）
  - 主动中断（abort）
  - 设置读取/保存/连接测试
  - 错误态文案与 UI 反馈

## 二、可增加功能（建议分阶段推进）

### 阶段 A：高价值、低到中等复杂度
1. 会话历史管理
- 能力：多会话切换、历史消息持久化、恢复上次会话。
- 价值：显著提升可用性和长期使用体验。

2. 连接状态与诊断面板
- 能力：展示连接状态、最近错误、平均延迟、重连状态。
- 价值：降低排障成本，适合直播场景快速定位问题。

3. 国际化（i18n）
- 能力：中英切换，文案集中管理。
- 价值：方便后续开源协作和跨语言用户。

### 阶段 B：中到高复杂度，体验增强明显
4. 语音交互（VAD）
- 能力：自动语音检测、自动发送语音输入、误触发抑制。
- 价值：从“文本控制”升级为“自然口语交互”。

5. 主动发言（Proactive Speak）
- 能力：空闲计时触发角色主动输出。
- 价值：直播陪伴感更强，角色更“活”。

6. 多模态输入（摄像头/截图）
- 能力：截图或摄像头帧输入视觉接口，结合文本问答。
- 价值：拓展玩法（看图评论、屏幕讲解、互动解说）。

## 三、建议实施顺序（4 周示例）

### Week 1
1. 修复 output-handler 超时机制
2. 调整 Token 安全存储
3. 修复 SSE CRLF 兼容

### Week 2
1. 补关键链路自动化测试
2. 统一 README / AGENTS / docs 文档描述
3. 新增连接状态诊断信息

### Week 3
1. 上线会话历史（本地持久化）
2. 增加基础 i18n 框架（中/英）

### Week 4
1. 接入 VAD（实验开关）
2. 设计并验证主动发言策略
3. 评估多模态输入（截图优先）

## 四、验收标准（建议）
1. 稳定性
- 异常场景（上游无响应、网络抖动、主动中断）均可在 UI 可感知并自动释放资源。

2. 安全性
- Token 不再明文存储在普通配置文件。

3. 可维护性
- 文档可直接指导新开发者在 15 分钟内跑通主链路。

4. 测试质量
- 前端关键流式交互和设置功能具备自动化回归测试。
